# Grocery Store Management Project

## Project Overview
The **Grocery Store Management Project** focuses on managing and analyzing data related to daily grocery store operations. It includes information about products, inventory, suppliers, customers, employees, and customer orders. By organizing this data effectively, the system helps simulate how real grocery stores manage their operations and transactions.

This project demonstrates how structured databases help businesses maintain accurate records and support decision-making processes.

---

## Project Objectives
The main objectives of this project are:

- Design a **database system** for managing grocery store operations  
- Store and manage data related to **products, suppliers, customers, employees, and orders**  
- Perform **data retrieval and analysis using SQL queries**  
- Generate insights that help improve store management and operational efficiency  

---

## Key Features
This project enables the grocery store to:

- Track **product sales and overall store revenue**
- Monitor **inventory levels and product availability**
- Maintain records of **suppliers and product categories**
- Analyze **customer purchasing behavior**
- Manage **employee activities and store transactions**

---

## Database Components
The database consists of several entities that interact with each other:

- **Products** – Contains information about grocery items available in the store  
- **Suppliers** – Stores details about vendors supplying the products  
- **Customers** – Maintains records of customers purchasing items  
- **Orders** – Tracks purchase transactions made by customers  
- **Employees** – Stores information about store staff  
- **Categories** – Classifies products into different groups  

---

## Technologies Used
- **SQL** – For database creation, querying, and data analysis  
- **Relational Database Concepts** – Tables, Primary keys, Foreign keys, and Joins  

---

## Purpose of the Project
The purpose of this project is to demonstrate how SQL can be used to **store, manage, and analyze business data** in the retail domain. It highlights how structured databases support efficient store operations and enable data-driven decision making.
